位运算

前言

一直以来,对我来说位运算都是非常高深的东西
最近在补《JAVA编程思想》时想要透彻的学习好这门已经学习了很久的语言

于是我选择不避开难理解的概念
但当我深入了解后发现，其实也就是很简单的加减乘除罢了

运算规则

运算规则很简单

这里指的双边意思为，两个进行比较的位运算

& 当双边为1时为1
| 当双边有一个为1时则为1
^ 当双边相同时为0，不同时为1
~ 取反,0变1,1变0
<< 左位移x位
>> 右位移x位

参考资料
https://www.runoob.com/w3cnote/bit-operation.html

实际运用

只看运算和概念还是很模糊
刚好我开发MOD时发现

游戏内的字体是由一个4字节的int作为颜色的

部分代码

this.red = (float)(color >> 16 & 255) / 255.0F;
this.blue = (float)(color >> 8 & 255) / 255.0F;
this.green = (float)(color & 255) / 255.0F;
this.alpha = (float)(color >> 24 & 255) / 255.0F;
setColor(this.red, this.blue, this.green, this.alpha);
this.posX = x;
this.posY = y;
this.renderStringAtPos(text, dropShadow);

从这里可以看出
将四个字节分成了四个部分

1字节有8个位，也就是 $2^8=256$ 位数

所以前三个字节被拿来当作RGB三原色的数字 前三个颜色分别是G,B,R
怎么判断出来的呢？

使用一个浅粉色,十六进制为#FFB6C1,十进制为16758465
二进制为111111111011011011000001

比如第一个字节中
this.green = (float)(color & 255) / 255.0F;
使用&运算后
111111111011011011000001
000000000000000011111111
最后剩下
000000000000000011000001
也就是说，前面的部分就直接为0了
这时候转换成10进制就是193
为什么最后要用193/255呢？因为在这里需要的是0到1的一个浮点数

第一个字节很好理解
后续右位移的字节呢？
其实右位移也很简单
右位移多少位就向左边填充多少个0
右边多出来的部分直接删除

例如

111111111011011011000001
>> 8
000000001111111110110110
后面的运算跟第一个字节雷同
最后算出一个0到1的浮点数

修改透明度

现在已经知道了运算规则,如何修改透明度
透明度位于24个字节之后
也就是说
我先填写好透明度的数值,然后向左位移24位最后相加即可

也就是

    int color = (0xfff0f5 | (alpha << 24));

到此大功告成!
